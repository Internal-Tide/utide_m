solve_m
=======

.. autofunction:: utide.solve_m

`solve_m` 是 `solve` 的高效变体，专门针对具有相同时间向量的多个时间序列进行了优化。

主要区别
--------

* 它将天文计算与调和分析解算分开，从而在处理多个具有相同时间点的时间序列时（如模型数据）具有更高的效率。
* 它可以利用缓存的天文修正量 (`fuv_cache`) 和频率信息 (`freqs`) 来显著加快计算速度。

参数详解
--------

* **fuv_cache**: (F, U, V, lind) 的元组，由天文修正计算生成。
* **freqs**: 线性化后的频率信息。

其他参数与 :func:`utide.solve` 保持一致。
